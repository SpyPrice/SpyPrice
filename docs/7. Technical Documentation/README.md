# Техническая документация

Этот раздел — **единый источник правды** по архитектуре и контрактам сервиса (backend/API/ETL/БД) для команды. Требования к продукту задаются [`ТЗ.md`](../../ТЗ.md); здесь — согласованная техническая реализация.

## Как устроена папка
- `01_Architecture Overview.md` — компоненты системы и границы ответственности.
- `API.md` — публичные (для фронта) и служебные эндпоинты, модели запросов/ответов, ошибки.
- `Database.md` — ER‑схема (текстом), ключевые поля/связи/индексы и правила хранения.
- `Parser Architecture.md` — ETL/парсинг: адаптеры, планировщик, пайплайн, ошибки и ретраи.
- `Contracts.md` — согласованные DTO между API, UI и ETL (и правила синхронизации при изменениях).

## Связность между документами (Obsidian)
В `docs/2. Sprint 1 (Weeks 1-3)/01_Concept and Architecture.md` есть wikilinks на `[[API]]`, `[[Database]]`, `[[Parser Architecture]]`, `[[Contracts]]`, `[[01_Architecture Overview]]` — они должны указывать на файлы **в этом каталоге**.

## Где участники размещают наработки
- **Лёша (backend)** обновляет `[[API]]` и `[[Database]]`:
  - меняет эндпоинты и контракты моделей (Pydantic) в `API.md`,
  - обновляет сущности/связи/индексы и текст ER в `Database.md`.
- **Эдик (ETL/parser)** обновляет `[[Parser Architecture]]`:
  - добавляет/меняет контракт извлечения цены и опциональных полей (например доступность),
  - описывает особенности адаптеров источников и обработку ошибок.
- **Егор (frontend)** согласует с Лёшей отражение DTO:
  - секция `Frontend types` в `API.md` и/или блоки в `[[Contracts]]`.
- **Рома (DevOps)** поддерживает в **корневом `README.md`** воспроизводимые инструкции (docker-compose, env, деплой); при смене переменных окружения для ETL/API — синхронно с Лёшей/Эдиком обновляет разделы запуска.

## Правило обновлений
Если вы меняете контракт между слоями (API или модель данных), делайте это так:
- сначала обновите документ “источника” (обычно `API.md` или `Database.md`),
- затем добавьте краткое примечание “что изменилось и где это отразится” в связанный документ (`Contracts.md` при затронутых DTO).
