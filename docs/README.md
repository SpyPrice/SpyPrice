# Документация проекта Price Tracker

Команда ведёт артефакты в Markdown (удобно открывать папку `price-tracker` как хранилище Obsidian). **Источник требований к продукту** — файл [`ТЗ.md`](../ТЗ.md) в корне репозитория; разделы ниже детализируют реализацию и процесс, но не заменяют официальное задание.

## Навигация по разделам

| Раздел | Содержание |
|--------|------------|
| [`1. Project Management/`](1.%20Project%20Management/) | Роли, глоссарий, решения (ADR), **программа кейса и контрольные точки** |
| [`2. Sprint 1 (Weeks 1-3)/`](2.%20Sprint%201%20(Weeks%201-3)/) | Цель спринта 1, концепция, user stories, индивидуальные планы, шаблон протокола встречи |
| [`7. Technical Documentation/`](7.%20Technical%20Documentation/) | Архитектура, API, БД, парсер/ETL, контракты между слоями |

## Обязательные артефакты по итогам кейса (из ТЗ)

По завершении предусмотрены:

1. Публично доступный работающий MVP.
2. Git-репозиторий с историей коммитов.
3. Техническая документация в **корневом `README.md`**: архитектура, стек, схема данных, сборщик, локальный запуск, поддерживаемые источники и ограничения.
4. Финальная презентация (10–15 слайдов) и защита.

Детали сроков, показов и критериев оценивания — в [`1. Project Management/Program and Milestones.md`](1.%20Project%20Management/Program%20and%20Milestones.md).

## Кто за что отвечает в документах

- **Общие процессные вещи** (роли, milestone’ы) — Team Lead и вся команда при согласовании.
- **Технические контракты** — см. [`7. Technical Documentation/README.md`](7.%20Technical%20Documentation/README.md) (зоны ответственности по файлам).

Папка `.obsidian/` не коммитится (см. корневой `README.md`), чтобы настройки редактора не конфликтовали.
