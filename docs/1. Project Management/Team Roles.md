# Роли команды и зоны ответственности

Проект: **сервис отслеживания и анализа изменения цен в онлайн‑магазинах** (MVP).

## Фёдор — Team Lead / Project Manager + Дизайнер
- **Ответственность**: процесс и синхронизация команды, планирование спринтов, бэклог, подготовка показов; UI/UX, wireframes, прототипы.
- **Артефакты**: user stories + acceptance criteria, макеты/прототип (Figma), презентации, структура документации.

## Егор — Frontend-разработчик
- **Ответственность**: SPA-интерфейс, визуализация истории цен (графики/таблицы), интеграция с API, адаптивность и UX-состояния.
- **Артефакты**: код фронтенда, прототип интерфейса (в коде), согласованные интерфейсы данных.

## Лёша — Backend-разработчик (Python)
- **Ответственность**: архитектура сервера, API, бизнес‑логика, модель данных, авторизация (JWT), миграции, документация OpenAPI.
- **Артефакты**: схема БД (ER), спецификация API, каркас приложения, миграции, демо‑данные.

## Рома — DevOps / Инфраструктура
- **Ответственность**: окружение разработки/демо, Docker/Compose, CI (линтеры/сборка), базовый деплой стенда для показов.
- **Артефакты**: Dockerfile(ы), `docker-compose` (dev/prod), GitHub Actions, инструкции по запуску и деплою.

## Эдик — Data/ETL инженер
- **Ответственность**: источники данных, адаптеры парсинга, прототип сборщика, нормализация и сохранение PriceSnapshot, устойчивость к ошибкам.
- **Артефакты**: код ETL/парсера, описание источников/ограничений, протокол сохранения срезов (через API или БД).
