# Глоссарий

- **MVP**: минимально жизнеспособный продукт — набор функций, достаточный для демонстрации ценности.
- **User Story**: пользовательская история в формате «Как \<роль\>, я хочу \<цель\>, чтобы \<ценность\>».
- **Acceptance Criteria**: критерии приёмки — условия, при которых user story считается выполненной.
- **Backlog**: список задач/историй, упорядоченный по приоритету.
- **Sprint**: ограниченный по времени итерационный цикл разработки (у нас — блоки недель).
- **PriceSnapshot**: запись‑срез цены товара в момент времени (цена, валюта, fetched_at, статус и т.п.).
- **ETL**: Extract‑Transform‑Load — извлечение данных, преобразование/нормализация, загрузка в хранилище.
- **Source**: источник данных (магазин), из которого извлекаются цены.
- **Parser / Adapter**: модуль под конкретный источник (правила извлечения цены/наличия и обработка ошибок).
- **Scheduler**: планировщик периодических задач (cron/Celery beat/APScheduler).
- **CRUD**: Create‑Read‑Update‑Delete — базовые операции над сущностью.
- **JWT**: JSON Web Token — формат токена для аутентификации/авторизации.
- **OpenAPI/Swagger**: спецификация API и интерактивная документация.
- **CORS**: политика браузера, ограничивающая кросс‑доменные запросы; на бэкенде обычно настраивается список разрешённых источников.
